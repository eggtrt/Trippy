데이터 흐름 (Tool Use & MCP 기반)

  1. Client → Route 53을 통해 도메인 해석
  2. Route 53 → EC2 웹 서버로 요청 전달
  3. EC2 → Bedrock (Claude AI)에 사용자 요청 전달 (Tool 정의 포함)
  4. Bedrock → Tool Use를 통해 필요한 정보 판단 및 함수 호출 요청 반환
  5. EC2 → MCP 서버들에 Tool 실행 요청 (Google Maps, Open-Meteo, Currency Exchange, Travel MCP Server)
  6. MCP Servers → EC2로 정보 반환 (장소, 날씨, 환율, 항공/숙박 정보)
  7. EC2 → Tool 실행 결과를 Bedrock에 다시 전달
  8. Bedrock → 실제 데이터 기반으로 최종 여행 계획 생성 (채팅 메시지 + 구조화된 JSON)
  9. EC2 → Aurora에 사용자 데이터 및 여행 일정 저장
  10. EC2 → S3에 로그 및 정적 파일 저장
  11. EC2 → Client에 채팅 응답 및 구조화된 여행 계획 전달
